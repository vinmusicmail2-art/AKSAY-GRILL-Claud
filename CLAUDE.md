# Инструкции для Claude Code

## Правила работы с репозиторием

### Проверка источника при копировании файлов

**КРИТИЧЕСКИ ВАЖНО:** При создании нового репозитория или копировании файлов ВСЕГДА сравнивайте с оригинальным рабочим репозиторием на GitHub, а НЕ с локальной папкой.

**Причина:** Локальные папки могут содержать незакоммиченные изменения, эксперименты или устаревшие версии файлов.

**Процедура:**
1. Если существует рабочий репозиторий на GitHub - клонируйте его как эталон
2. Сравнивайте все критичные файлы (HTML, CSS, конфиги) с эталоном
3. Используйте `md5sum` или `diff` для побайтового сравнения
4. Проверяйте особенно внимательно:
   - `templates/*.html` - размеры шрифтов, стили, классы
   - `assets/css/*.css` - все CSS правила
   - `tailwind.config.js` - конфигурация размеров и шрифтов
   - `.replit` - пути и команды запуска

**Пример ошибки:**
- Скопировал из локальной папки `aksay_gril_20260503_200418`
- В ней был изменён `font-size: 13px` вместо правильного `15px`
- Пользователь увидел изменение шрифтов, хотя я не менял код
- Решение: сравнил с GitHub репозиторием и нашёл различие

**Команда для проверки:**
```bash
# Клонировать эталон
git clone https://github.com/user/repo.git reference_repo

# Сравнить критичные файлы
md5sum reference_repo/templates/index.html new_repo/templates/index.html
diff -u reference_repo/assets/css/main.css new_repo/assets/css/main.css
```

## Работа с Git

### Создание чистого репозитория

При оптимизации репозитория (удаление истории):
1. Создать новую папку с `git init`
2. Скопировать файлы из **GitHub эталона**, не из локальной папки
3. Проверить все критичные файлы на идентичность
4. Только после проверки делать коммит и push

### Коммиты

- Всегда указывать `Co-Authored-By: Claude Sonnet 4 <noreply@anthropic.com>`
- Описывать ЧТО изменено и ПОЧЕМУ
- Для исправлений начинать с "Fix:"

## Рефакторинг кода

### Что можно менять без подтверждения

- Python backend код (routes, models, utils)
- Логику обработки данных
- Структуру БД (миграции)

### Что НЕЛЬЗЯ менять без явного запроса

- HTML шаблоны (структура, классы, inline стили)
- CSS файлы (размеры, шрифты, цвета)
- JavaScript код
- Конфигурационные файлы (.replit, tailwind.config.js)
- Файлы шрифтов

**Если пользователь жалуется на изменение внешнего вида:**
1. Немедленно сравнить с GitHub эталоном
2. Найти различия с помощью `diff` и `md5sum`
3. Восстановить оригинальные файлы
4. Объяснить, что именно было изменено

## Отладка проблем

### Пользователь говорит "ты изменил шрифты/размеры"

**Алгоритм:**
1. Клонировать оригинальный репозиторий с GitHub
2. Сравнить побайтово все HTML/CSS файлы
3. Найти различия с помощью `diff`
4. Если различия есть - восстановить из GitHub
5. Если различий нет - проблема в кэше браузера или Replit

**НЕ спорить** с пользователем. Сначала проверить факты.
