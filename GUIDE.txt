▌ 1. Вход в административную панель
────────────────────────────────────────────────────────────

Адрес: ваш-домен.ru/admin

При первом переходе в /admin система перенаправит вас на страницу входа
/admin/login.

Поля для входа:
  • Логин — имя пользователя, заданное при первоначальной настройке
  • Пароль — пароль администратора

  ⚠ Первый запуск: Если вы открываете сайт впервые и администратор ещё
не создан, система откроет страницу `/admin/setup`. Введите логин и
пароль для создания первого администратора. После создания страница
setup больше не будет доступна.

После входа вы попадёте на дашборд. Для выхода — кнопка «Выйти» в верхнем
меню.

▌ 2. Главная страница (Дашборд)

Адрес: /admin

На дашборде отображается:

  Блок  |  Описание
  Новые заказы доставки  |  Количество необработанных заказов из корзины
  Новые заявки бизнес-ланч  |  Количество необработанных запросов на бизнес-ланч

Блоки со счётчиками являются ссылками — нажмите на любой из них, чтобы
перейти к соответствующему разделу заказов.

▌ 3. Заказы доставки
────────────────────────────────────────────────────────────

Адрес: /admin/delivery-orders

Сюда поступают заказы, оформленные через корзину на главной странице сайта
(кнопка «Заказать доставку»).


  ◆ Что содержит каждый заказ

  Поле  |  Описание
  Имя  |  Имя клиента
  Телефон  |  Контактный номер
  E-mail  |  Электронная почта (если указана)
  Адрес доставки  |  Куда доставлять
  Состав заказа  |  Список блюд с количеством и ценами
  Итоговая сумма  |  Общая стоимость заказа
  Комментарий  |  Пожелания клиента
  Дата и время  |  Когда поступил заказ
  IP-адрес  |  Технический идентификатор клиента


  ◆ Как обработать заказ

  1. Позвоните клиенту по указанному телефону для подтверждения
  2. Уточните время доставки
  3. Нажмите кнопку «Отметить обработанным» — заказ переместится в
     раздел «Выполненные»
  4. При необходимости нажмите снова — статус вернётся обратно к «Новым»

  ⚠ Важно: Система только принимает заказы. Расчёт доставки и работа с
курьером происходит вне сайта — по телефону.

▌ 8. Редактирование текстов сайта
────────────────────────────────────────────────────────────

Адрес: /admin/texts

Здесь можно изменить тексты, отображаемые на сайте, без привлечения
разработчика.


  ◆ Доступные для редактирования тексты

#### Главная страница
  • Название сайта, слоган, текст в герое (заголовок, описание мясных и
    овощных блюд)
  • Текст кнопок CTA («Заказать доставку», «Посмотреть отзывы»)
  • Текст в подвале страницы (copyright)

#### Контакты в шапке
  • Адрес ресторана
  • Номер телефона
  • Часы работы

#### Реквизиты оператора (для политики конфиденциальности)
  • Название организации, ИНН, ОГРНИП, дата регистрации
  • Юридический адрес, налоговый орган, e-mail, телефон

#### Страницы О нас / Кейтеринг / Мероприятия / Бизнес-ланч
  • Подзаголовки и описания на каждой странице

#### Карта
  • Адрес, координаты (широта/долгота), масштаб карты


  ◆ Как редактировать

  1. Перейдите в /admin/texts
  2. Найдите нужное поле
  3. Отредактируйте текст
  4. Нажмите «Сохранить»

Изменения применяются на сайте немедленно.

▌ 9. Настройка e-mail уведомлений
────────────────────────────────────────────────────────────

Адрес: /admin/email-settings

При каждом новом заказе или заявке система может автоматически отправлять
уведомление на указанный e-mail.


  ◆ Настройки

  Параметр  |  Описание
  E-mail для уведомлений  |  Адрес, на который будут приходить оповещения о новых заявках
  Уведомления включены  |  Главный переключатель: Да / Нет


  ◆ Тестирование

На странице email-настроек есть кнопка «Отправить тестовое письмо».
Используйте её, чтобы убедиться, что письма доходят.


  ◆ SMTP-сервер (для технического специалиста)

Настройки SMTP задаются через переменные окружения на сервере (не через
интерфейс):

  Переменная  |  Значение
  SMTP_HOST  |  Адрес почтового сервера (например, smtp.yandex.ru)
  SMTP_PORT  |  Порт: 465 (SSL) или 587 (STARTTLS)
  SMTP_USER  |  Логин почтового ящика
  SMTP_PASSWORD  |  Пароль от почтового ящика
  SMTP_FROM  |  E-mail отправителя (обычно = SMTP_USER)

  ⚠ Если SMTP не настроен — сайт работает в штатном режиме,
уведомления просто не отправляются. Ошибок для клиентов не возникает.

▌ 10. Безопасность и рекомендации
────────────────────────────────────────────────────────────


  ◆ Пароль администратора
  • Используйте надёжный пароль (минимум 12 символов, буквы + цифры +
    спецсимволы)
  • Не сообщайте пароль третьим лицам
  • При подозрении на компрометацию — смените пароль через /admin/setup
    (доступно только при удалении всех записей администраторов из базы
    данных, иначе — через разработчика)


  ◆ Работа с заказами
  • Обрабатывайте заказы как можно быстрее — клиенты ожидают обратной
    связи в течение 15–30 минут
  • Используйте кнопку «Обработан» только после того, как связались с
    клиентом
  • Обработанные заявки хранятся в системе и доступны для просмотра


  ◆ Сессия
  • Браузер автоматически разлогинивает вас при закрытии вкладки (если
    не выбрано «Запомнить»)
  • Всегда нажимайте «Выйти» при использовании общего компьютера


  ◆ Резервные копии
  • База данных хранится в файле data.db в папке сайта
  • Рекомендуется регулярно делать резервную копию этого файла

▌ 11. Что делать, если что-то пошло не так
────────────────────────────────────────────────────────────


  ◆ Не приходят e-mail уведомления
  1. Перейдите в /admin/email-settings
  2. Убедитесь, что уведомления включены и указан правильный e-mail
  3. Нажмите «Отправить тестовое письмо»
  4. Если письмо не пришло — проверьте папку «Спам»
  5. Если проблема сохраняется — обратитесь к разработчику для проверки
     SMTP-настроек


  ◆ Клиент говорит, что не смог отправить форму
  1. Попросите клиента обновить страницу и попробовать снова
  2. Уточните, какое сообщение об ошибке отображается
  3. Примите заказ по телефону вручную


  ◆ Счётчик на дашборде не уменьшается
  • Убедитесь, что вы нажали кнопку «Отметить обработанным» (а не просто
    просмотрели заявку)
  • Обновите страницу браузера


  ◆ Сайт не открывается
  • Обратитесь к хостинг-провайдеру или разработчику

▌ 12. Уведомление в РКН — соблюдение 152-ФЗ
────────────────────────────────────────────────────────────

Сайт собирает персональные данные посетителей (имя, телефон, e-mail, адрес
доставки). По Федеральному закону № 152-ФЗ «О персональных данных» вы
обязаны уведомить Роскомнадзор о том, что являетесь оператором персональных
данных. Без уведомления возможен штраф по ст. 13.11 КоАП РФ.

  ⚠ Важно: Уведомление нужно подать до начала обработки персональных
данных, то есть до публикации сайта в открытый доступ.


  ◆ Шаг 1 — Войдите в личный кабинет РКН

Перейдите на портал: https://pd.rkn.gov.ru

Авторизуйтесь через Госуслуги (ЕСИА). Для ИП — вход через личный кабинет
физлица с ИП-статусом; для ООО — через руководителя организации.


  ◆ Шаг 2 — Заполните уведомление

Раздел: «Реестр операторов» → «Подать уведомление»

Заполните форму, используя данные ниже как ориентир для вашей организации:

  Поле формы  |  Что указывать
  Наименование оператора  |  Полное наименование ИП или ООО
  ИНН  |  ИНН организации/ИП
  Адрес  |  Юридический адрес
  Цели обработки ПД  |  Обработка заказов на доставку еды; организация мероприятий и кейтеринга; связь с клиентами по вопросам заказов
  Категории ПД  |  Фамилия, имя; номер телефона; адрес электронной почты; адрес доставки
  Категории субъектов  |  Клиенты (покупатели), посетители сайта
  Перечень действий  |  Сбор, запись, хранение, использование, передача (при доставке), уничтожение
  Правовое основание  |  Согласие субъекта ПД (ст. 6, ч. 1, п. 1 152-ФЗ)
  Меры защиты  |  Ограничение доступа к данным, хранение на серверах в РФ, пароли администраторов
  Место хранения  |  Российская Федерация (обязательное требование — см. раздел 13)
  Передача за рубеж  |  Нет
  Срок хранения  |  До исполнения заказа + 3 года (для бухгалтерских целей) или до отзыва согласия


  ◆ Шаг 3 — Отправьте уведомление

После заполнения формы нажмите «Подписать и отправить» (потребуется
электронная подпись через Госуслуги или КЭП).

Уведомление рассматривается в течение 30 дней. После рассмотрения вы
появитесь в реестре операторов ПД на сайте РКН.


  ◆ Что ещё нужно сделать помимо уведомления

  Требование  |  Статус на сайте
  Политика конфиденциальности  |  ✅ Размещена на сайте (/privacy.html)
  Согласие на обработку ПД  |  ✅ Чекбокс в формах заказов
  Уведомление об использовании cookies  |  ✅ Баннер на главной странице
  Хранение данных на серверах в РФ  |  ⚠️ Требует правильного выбора хостинга (см. раздел 13)
  Уведомление РКН  |  ❗ Необходимо подать самостоятельно


  ◆ Сроки и штрафы

  • Нарушение — до 100 000 руб. штрафа для ИП/организации по ст. 13.11
    КоАП
  • Повторное нарушение — до 500 000 руб.
  • С 1 сентября 2024 года ответственность существенно ужесточена

  ⚠ Рекомендация: Если не уверены в правильности заполнения —
обратитесь к юристу, специализирующемуся на защите персональных
данных. Стоимость разовой консультации обычно 3 000–8 000 руб., что
значительно дешевле штрафа.

▌ 13. Рекомендации по хостингу
────────────────────────────────────────────────────────────

По 152-ФЗ (ст. 18, ч. 5) персональные данные граждан РФ обязаны храниться
на серверах, физически расположенных на территории России. Это означает,
что сайт должен работать на российском хостинге.


  ◆ Рекомендуемые хостинг-провайдеры

  Провайдер  |  Тарифы (ориентир)  |  Особенности
  Beget (beget.com)  |  от 149 руб./мес.  |  Отличная поддержка, простая панель, подходит для Flask
  Timeweb (timeweb.com)  |  от 179 руб./мес.  |  Удобный интерфейс, быстрые VPS
  REG.RU (reg.ru)  |  от 149 руб./мес.  |  Крупнейший российский регистратор доменов + хостинг
  Selectel (selectel.ru)  |  от 350 руб./мес.  |  Профессиональный облачный хостинг, VPS/VDS
  Yandex Cloud (cloud.yandex.ru)  |  от 500 руб./мес.  |  Высокая надёжность, гибкое масштабирование


  ◆ Что выбрать для данного сайта

Сайт написан на Python/Flask и требует VPS (виртуального сервера) — обычный
виртуальный хостинг (shared hosting) не подходит.

Оптимальный вариант для начала: Beget VPS или Timeweb VPS

Минимальные требования к серверу:
  • CPU: 1 ядро
  • RAM: 1 ГБ (рекомендуется 2 ГБ)
  • Диск: 10 ГБ SSD
  • ОС: Ubuntu 22.04 LTS
  • Стоимость: от 350–600 руб./мес.


  ◆ Домен

Зарегистрируйте домен в зонах .ru или .рф у российского регистратора
(REG.RU, Beget, Timeweb). Стоимость: от 199–799 руб./год в зависимости от
зоны.


  ◆ SSL-сертификат (HTTPS)

Обязателен для сайта, собирающего персональные данные. Варианты:
  • Let's Encrypt — бесплатно, автопродление, предоставляется
    большинством российских хостингов автоматически
  • Платный сертификат — от 1 500 руб./год (нужен для юридических лиц в
    некоторых случаях)


  ◆ Перенос сайта на российский хостинг

При переносе потребуется помощь разработчика для:
  1. Настройки сервера (Python, gunicorn, nginx)
  2. Настройки переменных окружения (SESSION_SECRET, SMTP и др.)
  3. Переноса базы данных (data.db)
  4. Привязки домена и выпуска SSL

▌ 14. Настройка e-mail для получения заказов
────────────────────────────────────────────────────────────

Сайт отправляет уведомления о новых заказах через SMTP-сервер. Ниже —
пошаговые инструкции для наиболее распространённых российских почтовых
сервисов.


  ◆ Вариант А — Яндекс 360 (рекомендуется)

Яндекс 360 позволяет подключить корпоративную почту на вашем домене
(например, zakazy@aksaygril.ru). Бесплатный тариф — до 5 пользователей.

Шаг 1. Зарегистрируйтесь на 360.yandex.ru и добавьте ваш домен.

Шаг 2. Подтвердите владение доменом (добавьте TXT-запись в DNS — хостинг-
провайдер поможет).

Шаг 3. Создайте почтовый ящик (например, zakazy@aksaygril.ru).

Шаг 4. Включите доступ по паролю для приложений:
  • Перейдите в настройки ящика → Безопасность → Пароли приложений
  • Создайте пароль приложения (не обычный пароль!)

Шаг 5. Передайте разработчику следующие настройки для .env или переменных
окружения:


    SMTP_HOST    = smtp.yandex.ru
    SMTP_PORT    = 465
    SMTP_USER    = zakazy@aksaygril.ru
    SMTP_PASSWORD = (пароль приложения из шага 4)
    SMTP_FROM    = zakazy@aksaygril.ru


────────────────────────────────────────────────────────────


  ◆ Вариант Б — Mail.ru для бизнеса

Аналогично Яндексу, Mail.ru предоставляет корпоративную почту для домена.

Шаг 1. Зарегистрируйтесь на biz.mail.ru, добавьте домен.

Шаг 2. Создайте ящик и включите двухфакторную аутентификацию.

Шаг 3. Создайте пароль для внешних приложений в настройках безопасности.

Настройки SMTP:


    SMTP_HOST    = smtp.mail.ru
    SMTP_PORT    = 465
    SMTP_USER    = zakazy@aksaygril.ru
    SMTP_PASSWORD = (пароль приложения)
    SMTP_FROM    = zakazy@aksaygril.ru


────────────────────────────────────────────────────────────


  ◆ Вариант В — Обычный Яндекс / Gmail (для тестирования)

Подходит только для начального этапа. Использует обычный почтовый ящик без
привязки к домену.

Яндекс.Почта:
  1. Войдите в ящик → Настройки → Почтовые программы
  2. Включите «Доступ по паролю приложения»
  3. Создайте пароль приложения


    SMTP_HOST    = smtp.yandex.ru
    SMTP_PORT    = 465
    SMTP_USER    = ваш-ящик@yandex.ru
    SMTP_PASSWORD = (пароль приложения)
    SMTP_FROM    = ваш-ящик@yandex.ru


────────────────────────────────────────────────────────────


  ◆ Как применить настройки SMTP

После получения настроек передайте их разработчику — он внесёт их в
переменные окружения сервера. Самостоятельно через сайт SMTP-параметры не
меняются (только e-mail получателя и включение/выключение уведомлений — в
/admin/email-settings).

После применения настроек:
  1. Перейдите в /admin/email-settings
  2. Нажмите «Отправить тестовое письмо»
  3. Убедитесь, что письмо пришло на указанный адрес


  ◆ На какой e-mail приходят уведомления о заказах

На адрес, указанный в поле «E-mail для уведомлений» в разделе /admin/email-
settings. Это может быть любой ящик — не обязательно тот, с которого
отправляется письмо. Например: SMTP-отправитель — zakazy@aksaygril.ru, а
получатель уведомлений — direktor@aksaygril.ru или личный телефонный ящик.


  ◆ Что содержит письмо-уведомление

При каждом новом заказе на указанный e-mail приходит письмо с полными
данными заявки: именем клиента, телефоном, адресом, составом заказа и
суммой. Письмо приходит в течение 1–2 минут после оформления заказа
клиентом.

▌ Приложение: Перенос сайта на российский VPS (для разработчика)
────────────────────────────────────────────────────────────

  ⚠ Этот раздел предназначен для технического специалиста,
выполняющего перенос сайта с Replit на российский VPS-сервер.

В папке deploy/ проекта находятся все необходимые файлы:


    deploy/
    ├── install.sh          — скрипт автоматической установки
    ├── nginx.conf          — конфигурация веб-сервера nginx
    ├── aksaygril.service   — systemd-сервис для автозапуска приложения
    └── requirements.txt    — список Python-зависимостей



  ◆ Требования к серверу

  • ОС: Ubuntu 22.04 LTS
  • CPU: 1 ядро (рекомендуется 2)
  • RAM: 1 ГБ (рекомендуется 2 ГБ)
  • Диск: 10 ГБ SSD
  • Открытые порты: 22 (SSH), 80 (HTTP), 443 (HTTPS)


  ◆ Шаг 1 — Скачать файлы проекта на сервер

Подключитесь к серверу по SSH и скачайте проект с Replit (или передайте
архивом по SCP):


    # Вариант А — клонировать из git-репозитория (если подключён)
    git clone https://github.com/ваш-репозиторий/aksaygril.git /tmp/aksaygril
    
    # Вариант Б — передать архив с локальной машины
    scp -r ./aksaygril root@ВАШ_IP:/tmp/aksaygril


Перейдите в папку проекта:

    cd /tmp/aksaygril



  ◆ Шаг 2 — Запустить скрипт установки


    bash deploy/install.sh


Скрипт автоматически:
  • Установит Python 3.11, nginx, certbot
  • Создаст системного пользователя aksaygril
  • Скопирует файлы в /var/www/aksaygril/
  • Создаст виртуальное окружение и установит все зависимости
  • Настроит nginx и зарегистрирует systemd-сервис


  ◆ Шаг 3 — Заполнить переменные окружения

Откройте файл сервиса:

    nano /etc/systemd/system/aksaygril.service


Заполните значения в блоке Environment=:

  Переменная  |  Что поставить
  SESSION_SECRET  |  Случайная строка 32+ символа (сгенерировать: python3 -c "import secrets; print(secrets.token_hex(32))")
  SMTP_HOST  |  smtp.yandex.ru
  SMTP_PORT  |  465
  SMTP_USER  |  Почтовый ящик отправителя
  SMTP_PASSWORD  |  Пароль приложения (не обычный пароль!)
  SMTP_FROM  |  Тот же ящик, что и SMTP_USER


  ◆ Шаг 4 — Запустить приложение


    systemctl daemon-reload
    systemctl start aksaygril
    systemctl status aksaygril


Убедитесь, что статус active (running).


  ◆ Шаг 5 — Выпустить SSL-сертификат

Убедитесь, что DNS домена уже указывает на IP сервера, затем:


    certbot --nginx -d aksaygril.ru -d www.aksaygril.ru


После успешного выпуска раскомментируйте HTTPS-блок в /etc/nginx/sites-
available/aksaygril и перезагрузите nginx:


    systemctl reload nginx



  ◆ Шаг 6 — Перенести базу данных

Если на сервере уже есть data.db с данными (заказами, текстами сайта):


    # Скопировать с Replit на сервер
    scp data.db root@ВАШ_IP:/var/www/aksaygril/data.db
    
    # Установить правильного владельца
    chown aksaygril:aksaygril /var/www/aksaygril/data.db
    chmod 640 /var/www/aksaygril/data.db
    
    # Перезапустить сервис
    systemctl restart aksaygril



  ◆ Полезные команды после установки


    # Просмотр логов в реальном времени
    journalctl -u aksaygril -f
    
    # Логи nginx
    tail -f /var/log/aksaygril/error.log
    tail -f /var/log/aksaygril/access.log
    
    # Перезапуск после обновления кода
    systemctl restart aksaygril
    
    # Обновление кода из git
    cd /var/www/aksaygril
    git pull
    systemctl restart aksaygril
    
    # Статус всех сервисов
    systemctl status aksaygril nginx



  ◆ Отличия от Replit

  Параметр  |  Replit  |  VPS
  Запуск  |  gunicorn --reload (авто-перезагрузка)  |  systemd (ручной рестарт при изменениях)
  Статика  |  Flask отдаёт /assets/  |  nginx отдаёт /assets/ напрямую (быстрее)
  Переменные окружения  |  Secrets в интерфейсе Replit  |  Environment= в .service файле
  База данных  |  data.db в корне проекта  |  /var/www/aksaygril/data.db
  Логи  |  Консоль Replit  |  /var/log/aksaygril/ + journalctl
  HTTPS  |  Автоматически  |  Настраивается через certbot
