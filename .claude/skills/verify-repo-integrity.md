# Verify Repository Integrity

Проверяет целостность репозитория, сравнивая критичные файлы с оригинальным источником на GitHub.

## Назначение

Находит непреднамеренные изменения в frontend коде (HTML/CSS/JS), которые могли возникнуть при:
- Копировании файлов из локальных папок
- Создании нового репозитория
- Оптимизации Git истории
- Слиянии веток

## Использование

```bash
/verify-repo-integrity <github-repo-url>
```

**Пример:**
```bash
/verify-repo-integrity https://github.com/vinmusicmail2-art/AKSAY-GRILL
```

## Что проверяется

1. **HTML шаблоны** (`templates/*.html`)
   - Inline стили (font-size, font-family, colors)
   - CSS классы
   - Структура разметки

2. **CSS файлы** (`assets/css/*.css`)
   - Все правила стилей
   - Размеры шрифтов
   - Цвета и отступы

3. **Конфигурационные файлы**
   - `tailwind.config.js` - настройки размеров и шрифтов
   - `.replit` - команды запуска и пути
   - `package.json` - зависимости

4. **Статические ресурсы**
   - Файлы шрифтов (`.woff2`)
   - Изображения (проверка наличия)

## Алгоритм работы

1. Клонировать reference репозиторий во временную папку
2. Сравнить контрольные суммы (md5sum) критичных файлов
3. Для различающихся файлов показать diff
4. Предложить восстановить оригинальные версии

## Выходные данные

**Если различий нет:**
```
✅ Все критичные файлы идентичны оригиналу
Проверено файлов: 45
```

**Если найдены различия:**
```
⚠️ Найдены различия в 2 файлах:

1. templates/index.html
   Строка 189: font-size: 13px → должно быть 15px
   
2. assets/css/main.css
   Строка 54: font-weight: 300 → должно быть 200

Восстановить оригинальные версии? (y/n)
```

## Автоматическое исправление

Если пользователь подтверждает (y), скилл:
1. Копирует оригинальные файлы из reference репозитория
2. Создаёт коммит с описанием исправлений
3. Предлагает запушить изменения

## Примечания

- Скилл НЕ проверяет Python backend код (только frontend)
- Временная папка с reference репозиторием удаляется после проверки
- Если GitHub репозиторий недоступен - скилл завершается с ошибкой
